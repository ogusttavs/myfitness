# Story 5.1 — Timer Engine + Tests

**Epic:** EPIC-5 Execution + Timer
**Status:** Done
**Owner:** @dev
**Refs:** spec.md §3 EPIC-5, complexity.json (CDD-01), research.json (R-01)

## Goal
Implementar o motor do cronômetro de descanso como classe TS pura, sem React e sem dependência de runtime nativo. Drift-free, totalmente testável.

## Acceptance Criteria
- [x] Classe `RestTimer` com API: start, pause, resume, adjust(±ms), skip, reset, restore, subscribe
- [x] Drift baseado em relógio absoluto (Date.now)
- [x] Hook `useTimer` envolve a classe e expõe snapshot reativo + callback `onFinish` único
- [x] Testes unitários cobrindo: start, pause, resume, adjust ±, skip, reset, drift, restore from snapshot, subscribe/unsubscribe
- [x] Scheduler injetável (testes usam fake clock)

## File List
- apps/mobile/src/lib/timer/RestTimer.ts
- apps/mobile/src/lib/timer/RestTimer.test.ts
- apps/mobile/src/lib/timer/useTimer.ts

## Dev Notes
- Engine não conhece notificações, som, haptic — isso é responsabilidade do consumer (story 5.3).
- `restore()` permite vir de MMKV após app voltar do background; finaliza imediatamente se tempo já passou.
- `adjust(-ms)` mantém ao menos 1s no remaining para não disparar fim acidental.
